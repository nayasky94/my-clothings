# 나의 옷장 MVP PRD

## 핵심 정보

**목적**: Notion을 CMS로 활용하여 보유 중인 옷을 체계적으로 정리하고 오늘의 착장을 기록한다
**사용자**: 자신의 옷을 디지털로 관리하고 매일 착장을 기록하고 싶은 1인 사용자

---

## 사용자 여정

```
1. 홈 페이지 (랜딩)
   - 최근 착장 카드 목록 표시
   ↓ 상단 네비게이션 클릭

2-A. 오늘의 착장 페이지
   - Notion 옷장 목록에서 오늘 입은 옷 선택
   - 선택 완료 → 홈 페이지로 이동 (최근 착장 목록 갱신)

2-B. 옷장 페이지
   - 전체 옷 목록 표시
   - 카테고리 필터 선택 → 해당 카테고리 옷 목록 표시
   - 검색어 입력 → 일치하는 옷 목록 표시

3. [완료]
   - 착장 기록 완료 → 홈 페이지에서 최근 착장 확인
   - 옷 탐색 완료 → 옷장 페이지에서 계속 탐색
```

## 기능 명세

### 1. MVP 핵심 기능

| ID       | 기능명              | 설명                                                       | MVP 필수 이유                 | 관련 페이지                     |
| -------- | ------------------- | ---------------------------------------------------------- | ----------------------------- | ------------------------------- |
| **F001** | Notion 옷 목록 연동 | Notion API를 통해 옷 데이터베이스에서 옷 목록을 가져온다   | 서비스의 핵심 데이터 소스     | 옷장 페이지, 오늘의 착장 페이지 |
| **F002** | 카테고리 필터링     | 상의/하의/아우터/신발 등 카테고리별로 옷 목록을 필터링한다 | 옷이 많을수록 탐색 효율 필수  | 옷장 페이지                     |
| **F003** | 옷 검색             | 옷 이름, 태그 기준으로 키워드 검색을 수행한다              | 빠른 옷 탐색을 위한 기본 기능 | 옷장 페이지                     |
| **F004** | 오늘의 착장 선택    | 옷장 목록에서 오늘 입은 옷을 다중 선택하여 착장을 기록한다 | 서비스의 핵심 사용 시나리오   | 오늘의 착장 페이지              |
| **F005** | 최근 착장 목록 표시 | 홈 화면에서 최근 기록된 착장 목록을 카드 형태로 표시한다   | 서비스 첫 화면의 핵심 콘텐츠  | 홈 페이지                       |

## Notion API 연동 가이드

### 환경 변수 설정

```env
NOTION_API_KEY=secret_xxxxxxxxxxxxxxxxxxxx
NOTION_DATABASE_ID=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

### Notion 통합(Integration) 생성 순서

1. https://www.notion.so/my-integrations 접속
2. 새 통합 생성 → API 키 발급
3. Notion 데이터베이스 페이지 → 우상단 ... → 연결 → 생성한 통합 연결
4. 데이터베이스 URL에서 Database ID 추출

### 옷 목록 조회 예시 (서버 컴포넌트)

```typescript
// src/lib/notion.ts
import { Client } from '@notionhq/client'

const notion = new Client({ auth: process.env.NOTION_API_KEY })

export async function getClothingList() {
  const response = await notion.databases.query({
    database_id: process.env.NOTION_DATABASE_ID!,
    filter: {
      property: 'Status',
      select: { equals: '보유중' },
    },
    sorts: [{ property: 'Published', direction: 'descending' }],
  })
  return response.results
}
```

## 구현 순서

1. **Notion 설정**: 데이터베이스 생성 → 통합 연결 → 환경 변수 설정
2. **API 연동**: `@notionhq/client` 설치 → `src/lib/notion.ts` 작성
3. **옷장 페이지**: 전체 목록 렌더링 → 카테고리 필터 → 검색 기능
4. **오늘의 착장 페이지**: 다중 선택 UI → 로컬 스토리지 저장 로직
5. **홈 페이지**: 로컬 스토리지에서 최근 착장 읽어 카드 렌더링
6. **공통**: 네비게이션 헤더 → 반응형 레이아웃 → 빈 상태 처리
